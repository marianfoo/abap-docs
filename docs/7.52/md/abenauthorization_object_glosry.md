authorization object

[Repository object](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenrepository_object_glosry.htm "Glossary Entry") on which [authorizations](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenauthorization_glosry.htm "Glossary Entry") are based. An authorization object consists of up to ten [authorization fields](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenauthorization_field_glosry.htm "Glossary Entry"). The combination of authorization fields that represent data and activities is used for [authorization assignments](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenauthorization_assign_glosry.htm "Glossary Entry") and [authorization checks](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenauthorization_check_glosry.htm "Glossary Entry"). Authorization objects are organized in authorization object classes and are edited using [transaction code](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abentransaction_code_glosry.htm "Glossary Entry") SU21.
