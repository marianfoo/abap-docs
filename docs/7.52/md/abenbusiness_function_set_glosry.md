business function set

Abbreviated as BFS. [Repository object](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenrepository_object_glosry.htm "Glossary Entry") that groups multiple [business functions](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenbusiness_function_glosry.htm "Glossary Entry") to form a unit that covers the demands of an industry solution within the scope of [Switch Framework](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenswitch_framework_glosry.htm "Glossary Entry"). For every [AS ABAP](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abensap_nw_abap_glosry.htm "Glossary Entry"), one business function set can be switched on. Within a business function set, it is guaranteed that all business functions are compatible with each other. Business function sets are created in [transaction](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abentransaction_glosry.htm "Glossary Entry") SFW3 and switched in transaction SFW5.
