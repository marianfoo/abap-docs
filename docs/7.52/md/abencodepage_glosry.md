code page

Mapping of selected characters ([character set](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abencharacter_set_glosry.htm "Glossary Entry")) to bit sequences. For example, Latin1 ([ISO-8859-1](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abeniso-8859_glosry.htm "Glossary Entry")) represents a code page that contains all the characters required for Western European languages, while a [Unicode character representation](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenunicode_char_represent_glosry.htm "Glossary Entry") can display all characters required world-wide. The way in which characters are represented in an [internal session](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abeninternal_session_glosry.htm "Glossary Entry") is determined by the environment code page of the [text environment](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abentext_environment_glosry.htm "Glossary Entry") in the internal session.Each code page supported by SAP is assigned an [SAP code page number](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abensap_code_page_number_glosry.htm "Glossary Entry") in the [database table](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abendatabase_table_glosry.htm "Glossary Entry") TCP00. See also [single code page system](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abensystem_codepage_glosry.htm "Glossary Entry").
