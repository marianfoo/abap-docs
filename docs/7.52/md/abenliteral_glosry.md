literal

[Unknown data object](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenunnamed_data_object_glosry.htm "Glossary Entry") defined by specifying a value in the [ABAP source code](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenabap_source_code_glosry.htm "Glossary Entry") of an [ABAP program](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenabap_program_glosry.htm "Glossary Entry") directly (see also [Literals in ABAP CDS](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abencds_f1_literal.htm)). Literals can be either [numeric literals](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abennumeric_literal_glosry.htm "Glossary Entry") or [character literals](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abencharacter_literal_glosry.htm "Glossary Entry"). A [literal operator](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenliteral_operator_glosry.htm "Glossary Entry") enables multiple literals to be joined. See also [anonymous data object](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abenanonymous_data_object_glosry.htm "Glossary Entry") and [named data object](https://help.sap.com/doc/abapdocu_752_index_htm/7.52/en-US/abennamed_data_object_glosry.htm "Glossary Entry").
