UCS-2

Two byte [Unicode character representation](javascript:call_link\('abenunicode_char_represent_glosry.htm'\) "Glossary Entry"). In this representation, all characters are displayed using two bytes. The main difference to [UTF-16](javascript:call_link\('abenutf16_glosry.htm'\) "Glossary Entry") is that UCS-2 does not support characters from the [surrogate area](javascript:call_link\('abensurrogate_area_glosry.htm'\) "Glossary Entry"). The ABAP programming language supports UCS-2 in [Unicode systems](javascript:call_link\('abenunicode_system_glosry.htm'\) "Glossary Entry") and hence does not support [surrogates](javascript:call_link\('abensurrogates_glosry.htm'\) "Glossary Entry").
