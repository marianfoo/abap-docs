ABAP Source Code

The statements

-   [GENERATE SUBROUTINE POOL](javascript:call_link\('abapgenerate_subroutine_pool.htm'\)),

-   [READ REPORT](javascript:call_link\('abapread_report.htm'\)),

-   [SYNTAX-CHECK](javascript:call_link\('abapsyntax-check_for_itab.htm'\)),

-   [INSERT REPORT](javascript:call_link\('abapinsert_report.htm'\)),

for creating and editing [ABAP source ode](javascript:call_link\('abenabap_source_code_glosry.htm'\) "Glossary Entry") dynamically are released for use in application programming. Some additions of the released statements and certain statements are intended for internal system use only and must not be used in application programs.

Notes

-   The statements listed here can be used in application programs in exceptional cases only. ABAP provides many other means of dynamic programming, which generally make creating source code dynamically unnecessary (see the list in [dynamic program processing](javascript:call_link\('abenabap_language_dynamic.htm'\))).

-   If you want to display or edit source code in SAP GUI interactively, you can, for example, use the class CL\_GUI\_SOURCEEDIT. This class calls the ABAP Editor control.
