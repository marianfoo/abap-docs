batch input session

Groups a series of transaction calls together with input data and [user actions](javascript:call_link\('abenuser_action_glosry.htm'\) "Glossary Entry"). A batch input session can be used to execute [dialog transactions](javascript:call_link\('abendialog_transaction_glosry.htm'\) "Glossary Entry") in [batch input](javascript:call_link\('abenbatch_input_glosry.htm'\) "Glossary Entry"), where some or all of the [screens](javascript:call_link\('abenscreen_glosry.htm'\) "Glossary Entry") are processed by the session. Batch input sessions are saved persistently and are managed by the batch input manager in transaction SM35. See also [batch input tables](javascript:call_link\('abenbatch_input_table_glosry.htm'\) "Glossary Entry").
