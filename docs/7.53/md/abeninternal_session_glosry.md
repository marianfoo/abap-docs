internal session

Instance within an [ABAP session](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenmain_session_glosry.htm "Glossary Entry") with its own [program memory](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenroll_area_glosry.htm "Glossary Entry") in which the data and [objects](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenobject_glosry.htm "Glossary Entry") of an [ABAP program](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenabap_program_glosry.htm "Glossary Entry") are stored when the program is executed. An internal session is created when an ABAP program is called in the [shared memory](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenshared_memory_glosry.htm "Glossary Entry") and is preserved for as long as the [main program](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenmain_program_glosry.htm "Glossary Entry") of its [main program group](https://help.sap.com/doc/abapdocu_753_index_htm/7.53/en-US/abenmain_program_group_glosry.htm "Glossary Entry") is being executed .
