CDS entity

[CDS object](javascript:call_link\('abencds_object_glosry.htm'\) "Glossary Entry") in [ABAP CDS](javascript:call_link\('abenabap_cds_glosry.htm'\) "Glossary Entry") managed by [ABAP Dictionary](javascript:call_link\('abenabap_dictionary_glosry.htm'\) "Glossary Entry"). In a [CDS data definition](javascript:call_link\('abencds_data_definition_glosry.htm'\) "Glossary Entry"), a CDS entity is defined in [DDL source code](javascript:call_link\('abenddl_source_code_glosry.htm'\) "Glossary Entry") using the [CDS DDL](javascript:call_link\('abencds_ddl_glosry.htm'\) "Glossary Entry") in ABAP CDS. An instance of a CDS entity is created when it is activated. Within CDS, a CDS entity can access other entities. Currently, [CDS views](javascript:call_link\('abencds_view_glosry.htm'\) "Glossary Entry"), [CDS table functions](javascript:call_link\('abencds_table_function_glosry.htm'\) "Glossary Entry"), [CDS hierarchies](javascript:call_link\('abencds_hierarchy_glosry.htm'\) "Glossary Entry"), and [abstract CDS entities](javascript:call_link\('abenabstract_entity_glosry.htm'\) "Glossary Entry") are the available entities. In [ABAP programs](javascript:call_link\('abenabap_program_glosry.htm'\) "Glossary Entry"), a CDS entity can be used as a [data type](javascript:call_link\('abendata_type_glosry.htm'\) "Glossary Entry") and [ABAP SQL](javascript:call_link\('abenopen_sql_glosry.htm'\) "Glossary Entry") read statements can be used to access non-abstract CDS entities.
