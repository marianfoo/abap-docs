Literals

Alongside the [named data objects](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abennamed_data_object_glosry.htm "Glossary Entry") that can be addressed by their names in ABAP programs (this includes the text symbols) and the [anonymous data objects](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenanonymous_data_object_glosry.htm "Glossary Entry") created by the statement [CREATE DATA](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abapcreate_data.htm), there are also literals defined in the source code of programs by specifying a string that represents a value. Possible literals are:

-   [Numeric literals](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenuntyped_number_literals.htm)

-   [Character literals](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenuntyped_character_literals.htm)

The [literal operator](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenliteral_operator.htm) & can be used to associated multiple literals of the same type with each other.
