wildcard character

Character that used as a substitute for other content. In [relational expressions](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abencomparison_expression_glosry.htm "Glossary Entry") with the operators [CP](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenlogexp_strings.htm) and [NP](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenlogexp_strings.htm), "\*" usually stands for any [string](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abencharacter_string_1_glosry.htm "Glossary Entry") and "+" for any individual character. In [SQL conditions](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abensql_cond_glosry.htm "Glossary Entry") with the [operator](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenoperator_glosry.htm "Glossary Entry") [LIKE](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenwhere_logexp_like.htm), "%" stands for any string and "\_" for individual characters. In [regular expressions](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenregular_expression_glosry.htm "Glossary Entry"), a full set of [special characters](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenregex_syntax_specials.htm) is used to define patterns. The special functions of wildcard characters can be canceled using [escape characters](https://help.sap.com/doc/abapdocu_754_index_htm/7.54/en-US/abenescape_character_glosry.htm "Glossary Entry")
