parent-child relationship

PCR for short. A parent-child relationship is a directed [hierarchy edge](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenhierarchy_edge_glosry.htm "Glossary Entry") between two [hierarchy nodes](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenhierarchy_node_glosry.htm "Glossary Entry"), which defines the type of the nodes. A hierarchy node with outbound edges is a [parent node](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenparent_node_glosry.htm "Glossary Entry"). A hierarchy node with inbound edges is a [child node](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenparent_node_glosry.htm "Glossary Entry"). A hierarchy node with no inbound edges is a [root node](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenroot_node_glosry.htm "Glossary Entry") or an [orphan node](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenorphan_node_glosry.htm "Glossary Entry"). A hierarchy node with no outbound edges is a [leaf node](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenleaf_node_glosry.htm "Glossary Entry").
