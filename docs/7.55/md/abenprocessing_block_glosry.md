processing block

Modularization unit of an [ABAP program](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenabap_program_glosry.htm "Glossary Entry") that cannot be split or nested. Possible processing blocks are [procedures](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenprocedure_glosry.htm "Glossary Entry"), [dialog modules](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abendialog_module_glosry.htm "Glossary Entry"), and [event blocks](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenevent_block_glosry.htm "Glossary Entry"). They contain statements that are structured into [statement blocks](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abenstatement_block_glosry.htm "Glossary Entry") by [control structures](https://help.sap.com/doc/abapdocu_755_index_htm/7.55/en-US/abencontrol_structure_glosry.htm "Glossary Entry"). Each non-declarative statement in an ABAP program is part of a processing block.
