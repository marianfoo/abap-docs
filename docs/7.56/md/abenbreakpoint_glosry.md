breakpoint

Stopping point in an [ABAP program](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_program_glosry.htm "Glossary Entry") that when it is reached in [dialog processing](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abendialog_processing_glosry.htm "Glossary Entry"), is branched into the [ABAP Debugger](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_debugger_glosry.htm "Glossary Entry"). Breakpoints with a limited lifespan and a validity related to the current [ABAP user](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_user_glosry.htm "Glossary Entry") can be set interactively in the [ABAP Editor](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_editor_glosry.htm "Glossary Entry") or [ABAP Debugger](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_debugger_glosry.htm "Glossary Entry"). The statement [BREAK-POINT](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abapbreak-point.htm) can be used to create a breakpoint as an unconditional [checkpoint](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abencheckpoint_glosry.htm "Glossary Entry") with an unlimited lifespan, which is either always active or can be [activated](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenactivatable_checkpoint_glosry.htm "Glossary Entry") by assignment to a [checkpoint group](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abencheckpoint_group_glosry.htm "Glossary Entry"). .
