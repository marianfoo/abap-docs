client column

First column and [key field](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenkey_field_glosry.htm "Glossary Entry") of [client-dependent](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenclient_dependent_glosry.htm "Glossary Entry") [DDIC database tables](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenddic_db_table_glosry.htm "Glossary Entry") or [views](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenview_glosry.htm "Glossary Entry") of the type CLNT for storing the [client ID](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenclient_identifier_glosry.htm "Glossary Entry"). By default, [ABAP SQL](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_sql_glosry.htm "Glossary Entry") statements work with automatic [client handling](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenclient_handling_glosry.htm "Glossary Entry"). Database tables with a client column are generally [application tables](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenapplication_table_glosry.htm "Glossary Entry") but can also contain administrative data. Database tables without a client column are called [system tables](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abensystem_table_glosry.htm "Glossary Entry").
