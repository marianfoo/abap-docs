DCL

Short for Data Control Language. Subset of [SQL](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abensql_glosry.htm "Glossary Entry"). The statements of the DCL perform authorization and consistency checks on [relational databases](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenrelational_database_glosry.htm "Glossary Entry"). In Application Server ABAP, the functionality of the [DCL](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abencds_dcl_glosry.htm "Glossary Entry") is mapped using logical constructs such as [authorization objects](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenauthorization_object_glosry.htm "Glossary Entry") and [SAP locks](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abensap_lock_glosry.htm "Glossary Entry"). [ABAP CDS](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_cds_glosry.htm "Glossary Entry") provides its own [DDL](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abencds_dcl_glosry.htm "Glossary Entry"), which includes SQL and enhances it.
