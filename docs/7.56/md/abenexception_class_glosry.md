exception class

Special [class](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenclass_glosry.htm "Glossary Entry") that is the basis for [catchable](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abencatchable_exception_glosry.htm "Glossary Entry") exceptions. If an [exception](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenexception_glosry.htm "Glossary Entry") is raised, an [object](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenobject_glosry.htm "Glossary Entry") of an exception class is created. There are predefined exception classes for exceptions of the runtime framework, and self-defined exception classes for custom applications. The superclasses for all exception classes are CX\_STATIC\_CHECK, CX\_DYNAMIC\_CHECK, or CX\_NO\_CHECK, through which exceptions are divided into [exception categories](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenexception_category_glosry.htm "Glossary Entry").
