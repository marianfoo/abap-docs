pooled table

Before Release 7.53, [table category](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenddic_database_tables_tab_cat.htm) of a [DDIC database table](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenddic_db_table_glosry.htm "Glossary Entry") in the [ABAP Dictionary](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_dictionary_glosry.htm "Glossary Entry"). A pooled table was not defined as such on the [database](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abendatabase_glosry.htm "Glossary Entry"). Instead, multiple pooled tables were stored in a [table pool](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abentable_pool_glosry.htm "Glossary Entry") in the database. Only [ABAP SQL](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abenabap_sql_glosry.htm "Glossary Entry") could be used to access pooled tables whilst certain other restrictions also applied. Pooled tables are no longer supported since Release 7.53. Any existing pooled tables were transformed to [transparent tables](https://help.sap.com/doc/abapdocu_756_index_htm/7.56/en-US/abentransparent_table_glosry.htm "Glossary Entry").
