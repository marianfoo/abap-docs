character representation

Binary encoding of characters. [Character sets](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abencharacter_set_glosry.htm "Glossary Entry") are assigned to a character representation using [code pages](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abencodepage_glosry.htm "Glossary Entry"). The [ISO-8859](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abeniso-8859_glosry.htm "Glossary Entry") code pages are [single-byte](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abensingle_byte_code_glosry.htm "Glossary Entry") character representations, in which every character is encoded in up to one byte. SJIS and BIG5 are [double-byte](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abendouble_byte_code_glosry.htm "Glossary Entry") character representations for Japanese and traditional Chinese fonts. The [Unicode character representations](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenunicode_char_represent_glosry.htm "Glossary Entry") comprise all characters world-wide and can be encoded using one, two, or four bytes, depending on the [UTF](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenutf_glosry.htm "Glossary Entry") representation.
