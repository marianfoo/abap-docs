DDIC - Naming Rules for Database Tables

The name of a database table defined in the ABAP Dictionary must adhere to the following rules:

-   It can have a maximum of 16 characters.
-   It must follow the [naming rules for DDIC types](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenddic_types_names.htm).
-   It must follow the special [naming rules for DDIC structures](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenddic_structures_names.htm).
-   It is in the namespace of the [data types](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenddic_data_types.htm) in the ABAP Dictionary and hence in the namespace of all [global types](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenglobal_type_glosry.htm "Glossary Entry") of an AS ABAP.
