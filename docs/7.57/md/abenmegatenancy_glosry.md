multitenancy

Principle of software architecture in which a single software instance is run for different [tenants](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abentenant_glosry.htm "Glossary Entry") on a single server. In a multitenancy [AS ABAP](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenas_abap_glosry.htm "Glossary Entry"), [application clients](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenapplication_client_glosry.htm "Glossary Entry") assume the role of tenants. Multitenancy is not supported by the [ABAP runtime framework](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenabap_runtime_frmwk_glosry.htm "Glossary Entry") in the current release.
