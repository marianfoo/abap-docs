operand type

[Data type](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abendata_type_glosry.htm "Glossary Entry") with which an [operand position](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenoperand_position_glosry.htm "Glossary Entry") is typed. The operand type can be complete or generic. An [operand](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenoperand_glosry.htm "Glossary Entry") must match the operand type. Depending on the statement, [type conversions](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abentype_conversion_glosry.htm "Glossary Entry") or [castings](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abencast_casting_glosry.htm "Glossary Entry") take place for incompatible operands. In [inline declarations](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abeninline_declaration_glosry.htm "Glossary Entry"), the operand types determine the declared data type. For [constructor expressions](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenconstructor_expression_glosry.htm "Glossary Entry"), the operand type can be used to infer the type of the result.
