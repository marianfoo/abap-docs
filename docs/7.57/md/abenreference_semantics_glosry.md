reference semantics

Reference semantics means that the [reference](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenreference_glosry.htm "Glossary Entry") itself is used when accessing a [deep](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abendeep_glosry.htm "Glossary Entry") [data object](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abendata_object_glosry.htm "Glossary Entry"). In ABAP, reference semantics apply to [assignments](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenassignment_glosry.htm "Glossary Entry") and [comparisons](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abencomparison_glosry.htm "Glossary Entry") of [reference variables](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenreference_variable_glosry.htm "Glossary Entry") but not when accessing [dynamic data objects](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abendynamic_data_object_glosry.htm "Glossary Entry") or [field symbols](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenfield_symbol_glosry.htm "Glossary Entry"). See also [value semantics](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenvalue_semantics_glosry.htm "Glossary Entry").
