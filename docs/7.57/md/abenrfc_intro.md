RFC - Introduction

Remote Function Call (RFC) is an SAP-specific procedure for calling function modules in remote systems.

Communication using the RFC interface can be divided into different scenarios based on the systems involved:

-   AS ABAP - AS ABAP
-   AS ABAP - External system
-   AS ABAP - SAP Java
-   AS ABAP - external Java

RFC can be used as a synchronous or asynchronous communication procedure.

There are also different RFC variants that have been designed for different purposes and which therefore have different service properties:

-   [Synchronous RFC (sRFC)](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abensynchronous_rfc_glosry.htm "Glossary Entry")
-   [Asynchronous RFC (aRFC)](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenasynchronous_rfc_glosry.htm "Glossary Entry")
-   [Parallel RFC (pRFC)](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenparalel_remote_function_glosry.htm "Glossary Entry")
-   [Background RFC (bgRFC)](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenbg_remote_function_glosry.htm "Glossary Entry")
-   [Transactional RFC (tRFC), obsolete](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abentrfc_1_glosry.htm "Glossary Entry")
-   [Queued RFC (qRFC), obsolete](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenqueued_remote_function_glosry.htm "Glossary Entry")

Transactional and Queued RFC were replaced by Background RFC.
