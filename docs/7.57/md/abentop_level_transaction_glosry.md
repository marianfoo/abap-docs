top level transaction

The first program of a [call sequence](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abencall_sequence_glosry.htm "Glossary Entry"). The [ABAP memory](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenabap_memory_glosry.htm "Glossary Entry") assigned to a call sequence is retained for as long as the [internal session](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abeninternal_session_glosry.htm "Glossary Entry") of the top level transaction remains loaded. The initial [SAP Easy Access](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abensap_easy_access_glosry.htm "Glossary Entry") program does not usually work as a top level transaction, except in [area instance versions](https://help.sap.com/doc/abapdocu_757_index_htm/7.57/en-US/abenarea_instance_version_glosry.htm "Glossary Entry").
