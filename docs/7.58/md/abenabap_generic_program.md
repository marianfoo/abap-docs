ABAP Source Code

The statements

-   [GENERATE SUBROUTINE POOL](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abapgenerate_subroutine_pool.htm),
-   [READ REPORT](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abapread_report.htm),
-   [SYNTAX-CHECK](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abapsyntax-check_for_itab.htm),
-   [INSERT REPORT](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abapinsert_report.htm),

for creating and editing [ABAP source code](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenabap_source_code_glosry.htm "Glossary Entry") dynamically are released for use in application programming. Some additions of the released statements and certain statements are intended for internal system use only and must not be used in application programs.

Hints

-   The statements described here can be used in application programs in exceptional cases only. ABAP provides many other means of dynamic programming, which generally make creating source code dynamically unnecessary (see the list in [dynamic program processing](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenabap_language_dynamic.htm)).
-   To display or edit source code in SAP GUI interactively, the class CL\_GUI\_SOURCEEDIT can, for example, be used. This class calls the ABAP Editor control.
