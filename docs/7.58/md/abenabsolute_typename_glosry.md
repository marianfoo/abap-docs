absolute type name

[Type name](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abentype_name_glosry.htm "Glossary Entry"), constructed as a path in a [character string](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abencharacter_string_glosry.htm "Glossary Entry"), that uniquely specifies the [context](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenobj_context_glosry.htm "Glossary Entry") of a [data type](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abendata_type_glosry.htm "Glossary Entry"), a [class](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenclass_glosry.htm "Glossary Entry"), or an [interface](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenoo_intf_glosry.htm "Glossary Entry"), and which can be used in dynamic type specifications. The [RTTS](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenrun_time_type_services_glosry.htm "Glossary Entry") can determine the absolute type name of any [object](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenobject_glosry.htm "Glossary Entry"). See also [relative type name](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenrelative_type_name_glosry.htm "Glossary Entry").
