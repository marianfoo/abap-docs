Literals

In addition to the [named data objects](javascript:call_link\('abennamed_data_object_glosry.htm'\) "Glossary Entry") that can be addressed by their names in ABAP programs (this includes the text symbols) and the [anonymous data objects](javascript:call_link\('abenanonymous_data_object_glosry.htm'\) "Glossary Entry") that are created by the statement [CREATE DATA](javascript:call_link\('abapcreate_data.htm'\)), there are also [literals](javascript:call_link\('abenabap_literal_glosry.htm'\) "Glossary Entry") that are defined in the source code of a program by specifying a string that represents a value. Possible literals are:

-   [Numeric literals](javascript:call_link\('abenuntyped_number_literals.htm'\))
-   [Character literals](javascript:call_link\('abenuntyped_character_literals.htm'\))

The [literal operator](javascript:call_link\('abenliteral_operator.htm'\)) & can be used to link multiple literals of the same type to a literal.

Hint

In general ABAP, only [untyped literals](javascript:call_link\('abenuntyped_literal_glosry.htm'\) "Glossary Entry") are available. In [ABAP SQL](javascript:call_link\('abenabap_sql_glosry.htm'\) "Glossary Entry") [typed literals](javascript:call_link\('abentyped_literal_glosry.htm'\) "Glossary Entry") can and should be used as well.
