optimistic concurrency control

Optimistic [concurrency control](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenrap_conc_control_glosry.htm "Glossary Entry") enables transactional access to data by multiple users while avoiding inconsistencies by unintentional changes of already modified data. In [RAP](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenrap_glosry.htm "Glossary Entry") this is ensured by using an [ETag field](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenbdl_etag.htm) in the [RAP behavior definition](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abencds_behavior_definition_glosry.htm "Glossary Entry").
