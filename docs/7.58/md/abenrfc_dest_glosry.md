RFC destination

An RFC destination contains the connection parameters for [remote function calls](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenremote_function_call_glosry.htm "Glossary Entry"). It contains the connection type, the target system, and the partner program. Connections that only affect another [AS ABAP](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenas_abap_glosry.htm "Glossary Entry") can be classified as [trusted systems](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abentrusted_system_glosry.htm "Glossary Entry"). RFC destinations are managed statically using the [transaction](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abentransaction_glosry.htm "Glossary Entry") SM59 or dynamically using the class CL\_DYNAMIC\_DESTINATION.
