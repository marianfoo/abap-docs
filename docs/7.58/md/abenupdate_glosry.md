update

Execution of [update function modules](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenupdate_function_module_glosry.htm "Glossary Entry") that are registered using [CALL FUNCTION ... IN UPDATE TASK](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abapcall_function_update.htm) and triggered by [COMMIT WORK](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abapcommit.htm). Updates are used to bundle the changing database accesses of an [SAP LUW](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abensap_luw_glosry.htm "Glossary Entry") in a [database LUW](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abendatabase_luw_glosry.htm "Glossary Entry"). See also [synchronous](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abensynchronous_update_glosry.htm "Glossary Entry"), [asynchronous](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenasynchronous_update_glosry.htm "Glossary Entry"), and [local updates](https://help.sap.com/doc/abapdocu_758_index_htm/7.58/en-US/abenlocal_update_glosry.htm "Glossary Entry").
