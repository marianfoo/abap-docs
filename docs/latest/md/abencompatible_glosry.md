compatible

Except for [reference variables](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenreference_variable_glosry.htm "Glossary Entry"), two non- [generic data types](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abengeneric_data_type_glosry.htm "Glossary Entry") are compatible if all their [technical properties](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abentechnical_type_prpt_glosry.htm "Glossary Entry") are the same. Any [data type](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendata_type_glosry.htm "Glossary Entry") is compatible with a generic data type if the generic data type includes its technical properties. [Data objects](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendata_object_glosry.htm "Glossary Entry") are compatible if their data types are compatible. Compatibility is the basis for checking the [typing](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abentyping_glosry.htm "Glossary Entry"). In an [assignment](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenassignment_glosry.htm "Glossary Entry") or [comparison](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abencomparison_glosry.htm "Glossary Entry") of compatible data objects, no conversion takes place.
