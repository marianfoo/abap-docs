controlled SAP LUW

Extension of the [SAP LUW](javascript:call_link\('abensap_luw_glosry.htm'\) "Glossary Entry") concept. The controlled SAP LUW introduces a checking mechanism to detect transactional consistency violations. The controlled SAP LUW is automatically and implicitly supported by newer ABAP concepts such as [RAP](javascript:call_link\('abenrap_glosry.htm'\) "Glossary Entry"). In other contexts, the controlled SAP LUW can be used by explicitly setting transactional phases using the static methods of the class [CL\_ABAP\_TX](javascript:call_link\('abaprap_cl_abap_tx.htm'\)). In addition, [released APIs](javascript:call_link\('abenreleased_api_glosry.htm'\) "Glossary Entry") can have special classifications (starting with [IF\_ABAP\_TX...](javascript:call_link\('abapapi_classification.htm'\))) that enable specific transactional contracts that are checked in the [transactional phases](javascript:call_link\('abentr_phase_glosry.htm'\) "Glossary Entry").
