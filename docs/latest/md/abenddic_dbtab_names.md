DDIC - Naming Rules for Database Tables

The name of a database table defined in the ABAP Dictionary must adhere to the following rules:

-   It can have a maximum of 16 characters.
-   It must follow the [naming rules for DDIC types](javascript:call_link\('abenddic_types_names.htm'\)).
-   It must follow the special [naming rules for DDIC structures](javascript:call_link\('abenddic_structures_names.htm'\)).
-   It is in the namespace of [DDIC data types](javascript:call_link\('abenddic_data_types.htm'\)) and hence in the namespace of all [global types](javascript:call_link\('abenglobal_type_glosry.htm'\) "Glossary Entry") of an AS ABAP.
