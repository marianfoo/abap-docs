extended program check

Complete check of the [ABAP source code](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenabap_source_code_glosry.htm "Glossary Entry") for all statically recognizable errors. Programs with errors in the extended program check can be executed, but usually result in an [exception](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenexception_glosry.htm "Glossary Entry"). Called in the [ABAP Editor](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenabap_editor_glosry.htm "Glossary Entry") via Program → Check → Extended Program Check or via the [transaction code](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abentransaction_code_glosry.htm "Glossary Entry") SLIN or using the [ABAP Test Cockpit](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenabap_test_cockpit_glosry.htm "Glossary Entry"). Settings for the extended program check can be made in the administration transaction SLIN\_ADMIN.
