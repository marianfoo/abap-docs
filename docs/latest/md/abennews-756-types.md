Data Types in ABAP Release 7.56

Modification

Condensed Indicator Structures

The new addition [AS BITFIELD](abaptypes_indicators.htm#!ABAP_ADDITION_2@2@) behind addition [INDICATORS](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abaptypes_indicators.htm) of the statement [TYPES](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abaptypes.htm) allows a [condensed indicator structure](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abencondensed_ind_structure_glosry.htm "Glossary Entry") to be defined as a [byte field type](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenbyte_field_type_glosry.htm "Glossary Entry") component of a given [structured type](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenstructured_type_glosry.htm "Glossary Entry"). An condensed indicator structure can be used as a [null indicator](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abennull_indicator_glosry.htm "Glossary Entry") in ABAP SQL statements.
