protected

Term in [ABAP Objects](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenabap_objects_glosry.htm "Glossary Entry"). A protected [component](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenclass_component_glosry.htm "Glossary Entry") of a [class](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenclass_glosry.htm "Glossary Entry") can be accessed only by the class itself or by its subclasses. Only the class itself and its subclasses can create [objects](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenobject_glosry.htm "Glossary Entry") in the case of a protected instantiation. Here, [friends](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenfriend_glosry.htm "Glossary Entry") of the class have the same rights as the class itself. The protected [visibility section](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenvisibility_section_glosry.htm "Glossary Entry") of a class is declared in the statement [PROTECTED SECTION](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abapprotected.htm) in the [declaration section](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendeclaration_part_glosry.htm "Glossary Entry").
