reference

Content of [reference variables](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenreference_variable_glosry.htm "Glossary Entry"). A valid reference refers to a [data object](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendata_object_glosry.htm "Glossary Entry") ([data reference](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendata_reference_glosry.htm "Glossary Entry")) or an [instance](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abeninstance_glosry.htm "Glossary Entry") of a [class](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenclass_glosry.htm "Glossary Entry") ([object reference](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenobject_reference_glosry.htm "Glossary Entry")). References in reference variables are either [heap references](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenheap_reference_glosry.htm "Glossary Entry") or [stack references](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenstack_reference_glosry.htm "Glossary Entry"). Internally, [strings](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenstring_glosry.htm "Glossary Entry"), [internal tables](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abeninternal_table_glosry.htm "Glossary Entry"), and [boxed components](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenboxed_component_glosry.htm "Glossary Entry") are also addressed via references. From a technical point of view, an object in ABAP is referenced in multiple layers. A reference is the address of an internal header (object header, string header, or table header), which contains the address of the actual object and other administrative information such as the [data type](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendata_type_glosry.htm "Glossary Entry") of a referenced data object.
