Runtime Type Services

RTTS for short. Combination of [RTTC](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenrun_time_type_creation_glosry.htm "Glossary Entry") and [RTTI](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenrun_time_type_identific_glosry.htm "Glossary Entry") in a class hierarchy. The RTTS methods provide information on [data types](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendata_type_glosry.htm "Glossary Entry") or [classes](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenclass_glosry.htm "Glossary Entry") and [interfaces](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenoo_intf_glosry.htm "Glossary Entry") of [objects](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenobject_glosry.htm "Glossary Entry"), or they create new data types as [type description objects](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abentype_object_glosry.htm "Glossary Entry"). RTTS is implemented as a hierarchy of [type description classes](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abentype_class_glosry.htm "Glossary Entry") from which type description objects can be created.
