SPA/GPA parameter

Also referred to as SET/GET parameter. Data objects in the [user memory](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenuser_memory_glosry.htm "Glossary Entry") that can be accessed in [ABAP programs](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenabap_program_glosry.htm "Glossary Entry"). SPA/GPA parameters are set using [SET PARAMETER](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abapset_parameter.htm) and read using [GET PARAMETER](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abapget_parameter.htm). [Input fields](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abeninput_output_field_glosry.htm "Glossary Entry") on [dynpros](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendynpro_glosry.htm "Glossary Entry") can be linked with SPA/GPA parameters and can be prefilled with values when the dynpro is called. Moreover, user input in such fields remains stored in the corresponding SPA/GPA parameters. The names of SPA/GPA parameters are managed in the database table TPARA.
