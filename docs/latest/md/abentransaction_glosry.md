transaction

Execution of an [ABAP program](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenabap_program_glosry.htm "Glossary Entry") using a [transaction code](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abentransaction_code_glosry.htm "Glossary Entry"). There are [dialog](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendialog_transaction_glosry.htm "Glossary Entry"), [report](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenreport_transaction_glosry.htm "Glossary Entry"), [parameter](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenparameter_transaction_glosry.htm "Glossary Entry"), [variant](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenvariant_transaction_glosry.htm "Glossary Entry"), and [OO transactions](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenoo_transaction_glosry.htm "Glossary Entry"). A transaction is started by entering the transaction code in the [command field](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abencommand_field_glosry.htm "Glossary Entry") on the [standard toolbar](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenstandard_toolbar_glosry.htm "Glossary Entry") or by using the statements [CALL TRANSACTION](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abapcall_transaction.htm) or [LEAVE TO TRANSACTION](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abapleave_to_transaction.htm).

At SAP, database transactions are referred to as [database LUWs](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abendatabase_luw_glosry.htm "Glossary Entry").

In a business context, for example in [RAP](https://help.sap.com/doc/abapdocu_latest_index_htm/latest/en-US/abenrap_glosry.htm "Glossary Entry"), a transaction is refers to the result of a business-related operation that accesses and modifies data. When completed, a consistent state of data is produced.
